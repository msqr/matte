get-css-link

Type Named
Parameters
theme The ma:album-theme to render the URL for.
css The CSS resource path to use.
ua-type The browser type retured from get-browser-type.
ua-version The browser version retured from get-browser-version.
ua-platform The browser platform retured from get-browser-platform.
Result HTML <link> element
Requires
  1. get-resource-url
This template renders an HTML <link> element for a CSS resource, taking into account the browser type. For supported browser types this template will add a suffix to the CSS resource URL. This is to allow different CSS files based on different browser platforms.
Currently the only supported options are for Internet Explorer on Windows. If the ua-type is ie and the ua-platform is windows then a suffix of -ie-win will be appended before the assumed .css file name. Thus an input of browse.css would return browse-ie-win.css.

Example usage

<xsl:call-template name="get-css-link"> <xsl:with-param name="theme" select="$theme"/> <xsl:with-param name="css" select="$theme/ma:css"/> <xsl:with-param name="ua-type" select="$ua-type"/> <xsl:with-param name="ua-version" select="$ua-version"/> <xsl:with-param name="ua-platform" select="$ua-platform"/> </xsl:call-template>

Example output

/ma/theme/1/1029809801/browse-ie-win.css

Source

<xsl:template name="get-css-link"> <xsl:param name="theme"/> <xsl:param name="css"/> <xsl:param name="ua-type"/> <xsl:param name="ua-version"/> <xsl:param name="ua-platform"/> <link rel="stylesheet"> <xsl:attribute name="href"> <xsl:call-template name="get-resource-url"> <xsl:with-param name="theme" select="$theme"/> <xsl:with-param name="resource"> <xsl:choose> <xsl:when test="$ua-type = 'ie' and $ua-platform = 'win'"> <xsl:value-of select="substring-before($css,'.css')"/> <xsl:text>-ie-win.css</xsl:text> </xsl:when> <xsl:otherwise> <xsl:value-of select="$css"/> </xsl:otherwise> </xsl:choose> </xsl:with-param> </xsl:call-template> </xsl:attribute> </link> </xsl:template>
